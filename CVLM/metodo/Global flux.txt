step1

Le système reçoit un CV et une offre → il envoie à l’API LLM → il reçoit un texte → il génère un PDF.

step2
Entrée → récupération des fichiers (CV + offre)

Traitement → envoi à une API d’intelligence artificielle (LLM)

Sortie → réception du texte, génération d’un PDF final

step 3

| Étape              | Rôle                            | Responsabilité principale                        |
| ------------------ | ------------------------------- | ------------------------------------------------ |
| 1️⃣ Entrée         | Gérer les fichiers (CV + offre) | Lecture, extraction, validation                  |
| 2️⃣ Traitement     | Communiquer avec le LLM         | Formater et envoyer la requête, gérer la réponse |
| 3️⃣ Logique métier | Coordonner l’ensemble du flux   | Orchestration du processus complet               |
| 4️⃣ Sortie         | Générer le PDF final            | Mise en forme et export du document              |

step 4

FileManager → gère les fichiers d’entrée,

LLMClient → dialogue avec l’API,

PDFGenerator → crée le fichier PDF,

JobApplicationService → orchestre tout le flux.

Chaque classe correspond donc à une des étapes de ton flux.
